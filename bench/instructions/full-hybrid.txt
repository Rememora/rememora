## Rememora — MANDATORY Persistent Memory Protocol

You MUST use the `rememora` CLI for persistent memory. This is not optional.

### RULE 1: Search BEFORE implementing
Before writing ANY non-trivial code, run:
  rememora search "<topic>" --project express-api
Skip this only for trivial one-liners. If you implement without searching, you risk contradicting prior decisions.

### RULE 2: Save AFTER every significant action
After completing any task, you MUST save what you learned. Run one of:
  rememora save "chose X because Y" --category decision --project express-api --importance 0.8
  rememora save "fixed X: cause was Y" --category case --project express-api
  rememora save "convention: X" --category pattern --project express-api
  rememora save "service X at Y does Z" --category entity --project express-api

If you finish a task and have NOT run `rememora save`, stop and think about what you learned. There is almost always something to save.

### RULE 3: What NOT to save
Do not save: code content, git history, README content, temporary state.

### Why this matters
Your context is destroyed after this session. Other agents will work on this project next. If you don't save, knowledge dies with you. If you don't search, you'll duplicate work or contradict decisions.
